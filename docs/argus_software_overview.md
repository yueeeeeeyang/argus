# Argus 软件功能与优势介绍

## 1. 软件定位

Argus（阿尔戈斯）是一款面向开发者和运维人员的高性能桌面日志工具。它把本地日志阅读、压缩包浏览、跨文件搜索、Java 线程栈分析、Runtime 请求与 SQL 分析，以及 SSH/SMB/Git/SVN 远程操作整合到一个桌面工作台中，帮助用户缩短“获取日志—定位异常—分析根因”的完整链路。

项目使用 Rust 构建核心读取与分析能力，并使用 GPUI 渲染高信息密度桌面界面。当前包版本为 `0.1.0`，产品重点不是把日志一次性加载成普通文本，而是针对大文件、压缩日志和多文件分析采用不同的读取策略。

## 2. 核心价值

| 用户痛点 | Argus 的解决方式 | 带来的价值 |
| --- | --- | --- |
| 大日志打开慢、内存占用高 | 超过 30 MiB 的日志建立分块行索引，并按视口分页读取 | 大文件无需整体解码到内存，滚动和跳转更可控 |
| 日志散落在目录和多层压缩包中 | 统一来源树、压缩格式适配器、嵌套压缩包懒加载 | 减少手工解压、切换工具和重复查找文件的时间 |
| 单文件查找无法覆盖完整故障现场 | 支持当前文件、目录、选中文件三种搜索范围 | 一次搜索可以覆盖相关日志集合，并直接跳转到命中行 |
| 原始 Jstack 和 Runtime 日志难以快速理解 | 将线程快照、请求耗时和 SQL 明细转换成矩阵与聚合表格 | 从“阅读文本”升级为“比较趋势和定位热点” |
| 日志位于远程服务器、共享目录或代码仓库 | 内置 SSH 终端、SFTP、SMB 与 Git/SVN 只读文件管理 | 在同一应用内完成远程诊断、仓库浏览、文件预览和下载 |

## 3. 主要功能

### 3.1 多入口日志来源加载

Argus 支持通过以下方式加载本地文件、目录或压缩包：

- 内置来源选择器：支持常用位置、手动输入路径、多选，以及按名称或修改时间排序；
- 拖放到主窗口；
- 启动参数传入路径；
- 操作系统“用 Argus 打开”入口。Windows 使用当前用户注册表，macOS 使用 LaunchServices，可在设置中注册或卸载；
- 来源树内搜索、展开/收起和稳定选中定位。

目录与压缩包采用懒加载方式：只有用户展开节点时才读取子级，避免启动时递归扫描全部内容。来源注册表把真实路径、压缩包虚拟路径、父子关系与界面状态分离，使本地文件和压缩条目可以使用同一套打开、搜索和分析流程。

### 3.2 压缩包直接浏览与读取

当前代码内置以下压缩格式适配器：

| 格式 | 条目浏览与读取 | 密码处理 | 说明 |
| --- | --- | --- | --- |
| ZIP | 支持 | 支持 | 可识别文件头，支持嵌套压缩包 |
| 7Z | 支持 | 支持 | 可识别文件头，支持嵌套压缩包 |
| TAR | 支持 | 不适用 | 支持条目枚举和流式读取 |
| tar.gz / tgz | 支持 | 不适用 | 支持压缩 TAR 条目读取 |
| tar.bz2 / tbz2 | 支持 | 不适用 | 支持压缩 TAR 条目读取 |
| tar.xz / txz | 支持 | 不适用 | 支持压缩 TAR 条目读取 |
| GZIP 单文件 | 支持 | 不适用 | 单文件压缩包可直接作为日志打开 |
| RAR | 有条件支持 | 部分支持 | 不同 RAR 家族和加密算法的兼容性存在限制，应以实际文件验证为准 |

软件默认允许展开 2 层嵌套压缩包，并可在设置中调整，代码会把最大值限制在 8 层。单文件压缩包会折叠成可直接打开的叶子节点，减少无意义的树层级。

ZIP、7Z 和 RAR 等加密归档需要密码时会弹出输入框。压缩包密码只保存在当前进程内存中，不写入持久化配置；关闭应用后不会保留。

### 3.3 面向大文件的日志阅读

日志读取器根据来源和大小自动选择策略：

- 本地小文件：通过只读 mmap 获取字节并在内存中解码；
- 本地大文件：超过 30 MiB 后，按 8 MiB 数据块建立行号到字节偏移的索引，正文按可见行读取；
- 压缩包内小文件：从压缩条目顺序读取并在内存中展示；
- 压缩包内大文件：超过 30 MiB 后写入受控分页文件，再复用大文件行索引与按需读取能力；
- 分页日志使用有容量上限的解码行缓存，并对可见区域安排后台预取，减少滚动时的同步 I/O；
- 全文扫描按连续字节窗口合并读取，避免逐行频繁 `seek/read`。

读取器能处理 LF、CRLF、CR、UTF-16 换行和文件末尾无换行等情况。编码检测优先识别 BOM，再结合 UTF-8 校验和启发式检测；对 UTF-8、UTF-16LE/BE、GBK、GB18030、Big5 等常见日志编码提供检测或兜底解码，并在发现替换字符时尝试修复误判。

阅读区提供：

- 虚拟列表渲染、横纵滚动和动态行号栏；
- 日志、Java 线程栈、Properties/INI/CONF、XML、JSON、YAML 语法高亮；
- 鼠标字符/单词/整行选择和平台复制快捷键；
- 行号打点，以及通过 `F2` 循环跳转到下一个打点；
- 多标签页打开、切换、关闭当前/其他/全部标签。

### 3.4 多范围日志搜索

独立搜索窗口支持：

- 当前文件、指定目录、来源树选中文件三种范围；
- 普通关键字或 Rust 正则表达式；
- 大小写敏感切换；
- 多个预设“快搜关键字”一次扫描；
- 最近 20 条搜索关键字历史；
- 当前文件的命中计数、上一个/下一个快速定位；
- 文件数、行数和当前路径进度；
- 后台执行与主动取消；
- 结果按文件分组、全部展开/收起，并点击结果打开对应日志和定位行。

搜索引擎复用统一日志读取器，因此本地文件、压缩包条目和分页大日志使用一致的搜索语义。结果以批次回传，并限制界面刷新频率，避免高命中量搜索持续阻塞主界面。

### 3.5 Jstack 线程快照分析

Jstack 分析可以针对单个文件、多选文件、本地目录或压缩包中的日志发起。分析器增量解析线程块，并以“线程名 + 线程 ID 摘要”区分同名线程，最终生成跨快照频率矩阵。

主要能力包括：

- 识别 `RUNNABLE`、`BLOCKED`、`WAITING`、`TIMED_WAITING` 和其他状态；
- 按线程总出现次数聚合多个快照；
- 使用颜色矩阵对比线程在不同快照中的状态与出现次数；
- 按线程状态筛选；
- 按线程名子串、`*`/`?` 通配符或完整堆栈片段过滤低价值线程；
- 悬停预览堆栈，点击进入独立详情窗口；
- 在详情窗口切换同一线程的不同快照与重复出现记录，并复制选中的堆栈文本；
- 汇总成功快照、线程、样本和跳过文件数量，保留失败原因用于诊断。

这一能力适合发现持续阻塞线程、频繁出现的热点线程，以及比较多个时间点的线程状态变化。

### 3.6 Runtime 请求与 SQL 分析

Runtime 分析针对项目约定格式的请求日志，可递归读取本地目录中的 `.log` 文件，也可分析压缩包条目。大量文件会使用受控并行任务解析，当前并行上限为 8 个工作线程。

分析结果提供三类视图：

1. 请求统计：按请求地址聚合请求数量、平均耗时、慢 SQL 请求数量与比例；
2. SQL 频率分析：将 SQL 中的动态值归一化后按结构聚合，展示执行次数、总耗时和平均耗时，并可下钻到每次执行；
3. 慢 SQL 分析：当一个请求中 SQL 累积执行耗时超过请求总耗时的 90% 时，将其归入慢 SQL 请求，并按 SQL 结构聚合和下钻。

用户可以按任意关键字、一个或多个用户名、开始时间和结束时间组合过滤。过滤会同时作用于请求汇总、请求明细、SQL 频率和慢 SQL 结果，并通过结果缓存避免表格滚动时反复执行全量计算。SQL 原文可以在独立弹窗中完整查看和复制。

### 3.7 SSH 终端、远程文件与仓库管理

“链接”工作区可以使用目录组织 SSH、SMB、Git 或 SVN 链接，并支持新增、编辑、删除、过滤和折叠连接树。已有链接可直接拖放到任意目录，拖到树的空白区域则移回根层级；点击空白区域会取消当前树节点选择。远程文件标签使用实际协议名称（SFTP、SMB、Git 或 SVN）标识会话来源。

SSH 终端支持：

- 密码或私钥登录，私钥可配置口令；
- 首次连接时展示 SHA-256 主机指纹，由用户确认后记录受信指纹；
- `xterm-256color` PTY、窗口尺寸同步和 VT100 屏幕解析；
- 终端回滚、滚动条拖动、字符/单词/整屏选择与复制；
- 常见输入键、方向键以及 `Ctrl+C`、`Ctrl+D` 转发；
- 从终端右键直接打开对应 SFTP 文件管理。

SFTP 与 SMB 文件管理共享一套交互：

- 浏览目录、手动输入路径、返回父目录和刷新；
- 按名称、类型、大小、修改时间或权限排序；
- 单选或多选文件；
- 上传本地文件、下载单个或多个远程文件；
- 远程文件文本预览、当前目录内重命名、删除普通文件或空目录；
- 操作在后台 worker 中执行，连接失败与单次操作失败分开反馈。

远程预览只接受普通文件，默认文件大小不超过 2 MiB，单次最多读取 512 KiB；二进制内容会明确提示，避免把大文件或二进制文件直接载入界面。

Git 与 SVN 复用同一文件管理页面，但严格只读：可浏览目录、排序、刷新、预览普通文件、单文件或多文件下载，不显示也不接受上传、重命名和删除操作。Git 可切换默认分支、其他远程分支与标签；SVN 可输入 `HEAD` 或非负数字修订号。符号链接、Git 子模块、目录和其他非普通节点只展示类型，不跟随、不预览也不下载；Git LFS 内容按仓库中的指针文件读取，不调用 Git LFS。

Git 使用随应用构建的 `git2/libgit2` 完成 HTTPS 与 SSH 访问，不调用本机 `git` 或 `ssh`。每个链接在 `~/.argus/repositories/git/<link_id>.git` 维护无工作区的裸仓库缓存，刷新时 fetch/prune；远端暂时不可达且已有有效缓存时仍可浏览，并显示离线缓存警告。SVN 的 `svn://` 与 `svn+ssh://` 使用纯 Rust `svn` ra_svn 会话；`http://` 与 `https://` 使用内置 HTTPv2 DAV 客户端，通过 `OPTIONS`、`PROPFIND` 和 `GET` 直接读取修订资源，支持匿名或 HTTP Basic 用户名密码。两种实现都不创建 working copy 或持久内容缓存，也不依赖系统 Subversion、Cyrus SASL 或本机 `svn/ssh` 命令。

Git SSH 与 SVN SSH 都先展示 SHA-256 主机指纹并复用 Argus 受信主机流程；SVN 的 Rust SSH 栈禁用 agent、默认密钥和 OpenSSH 配置回退。Git HTTPS 和 SVN HTTPS 都使用操作系统信任根执行证书链及主机名校验，因此可识别已安装到系统证书库的企业 CA。`svn://` 与 `http://` 链路不加密，界面会分别提示改用 `svn+ssh://` 或 `https://`；DAV 凭据不会写入 URL、日志或错误响应正文，带认证请求只允许同源重定向并禁止 HTTPS 降级。

### 3.8 外观、配置与安全升级

- 主题使用 TOML 令牌定义，内置深色主题，也可从 `~/.argus/themes` 加载用户主题；
- 日志字号可在 12–20 px 范围调整；
- 来源加载策略、搜索偏好、Jstack 过滤规则、连接配置和升级配置统一持久化到 `~/.argus/settings.toml`；
- 自动升级默认关闭，只有配置服务器地址和 Ed25519 公钥后才会执行；
- 升级流程验证清单签名、平台/架构、SemVer、文件大小和 SHA-256；
- Windows/Linux 裸二进制与 macOS `.app` 使用不同的安全替换流程，macOS 会校验应用包结构后再替换和重启。

## 4. 软件优势

### 4.1 性能策略贯穿读取、搜索和渲染

Argus 的性能优势不是单一依赖 mmap，而是多层组合：大文件分块建索引、可见行按需读取、解码缓存、后台预取、虚拟列表、搜索批处理、事件合并和分析结果缓存。这样既控制内存，又减少主线程 I/O 和无效重绘。

### 4.2 压缩日志是一等来源

压缩包条目不是先由用户手工解压再导入，而是进入统一的来源、阅读、搜索与分析模型。嵌套压缩包懒加载、单文件压缩包折叠、密码上下文和超大条目自动分页，使常见“下载日志包后排障”的流程更直接。

### 4.3 从文本查找到结构化诊断

普通搜索解决“关键字在哪里”，Jstack 矩阵解决“线程状态如何变化”，Runtime 聚合解决“哪些请求和 SQL 最值得关注”。三层能力共享同一批日志来源，用户不需要在阅读器、脚本和表格工具之间反复搬运数据。

### 4.4 本地与远程排障入口统一

SSH 终端负责实时操作，SFTP/SMB 负责文件获取和管理，Git/SVN 负责代码仓库的只读取证，本地日志工作区负责深度阅读与分析。连接目录树、标签页和一致的文件管理交互降低了多服务器、多来源排障的切换成本。

### 4.5 具备明确的安全校验点

压缩包密码只驻留进程内存；SSH、Git SSH 和 SVN SSH 首次连接要求确认主机指纹；HTTPS Git 不放宽系统证书校验；升级包必须通过签名、哈希、大小和平台匹配校验。这些机制能够降低错误主机、篡改升级清单和损坏安装包带来的风险。

### 4.6 架构便于扩展

压缩格式通过适配器注册表扩展，日志来源与位置使用稳定 ID 和统一模型，远程文件管理通过能力模型和后端枚举复用 SFTP/SMB/Git/SVN 交互，语法高亮按语言规则拆分。新增格式、分析器或远程协议时，可以保持现有 UI 和业务入口稳定。

## 5. 典型使用场景

### 场景一：分析客户提供的压缩日志包

将 ZIP/7Z/RAR 或压缩 TAR 拖入 Argus，按目录树浏览或直接打开单文件压缩包；先用快搜同时检查 `ERROR`、`WARN` 等关键字，再点击结果定位上下文。遇到超大日志时，软件自动切换为分页读取。

### 场景二：比较多次 Java 线程快照

选择多个 Jstack 文件或包含快照的目录，发起 Jstack 分析；通过状态颜色和频率矩阵定位长期 `BLOCKED` 或反复出现的线程，再进入详情窗口比较每次堆栈。

### 场景三：定位请求和 SQL 性能问题

对 Runtime 日志目录发起分析，先按请求地址查看平均耗时和慢 SQL 比例，再切换到 SQL 频率或慢 SQL 视图，结合用户名与时间范围缩小问题窗口并下钻原始 SQL。

### 场景四：远程排障并获取日志

通过 SSH 终端执行诊断命令，再从终端打开 SFTP 文件管理，预览或下载日志；对于共享目录直接使用 SMB 链接，对 Git/SVN 仓库则选择分支、标签或修订号后只读浏览和下载。下载后的文件可继续在日志工作区搜索和分析。

## 6. 当前能力边界与使用注意事项

以下限制来自当前代码实现，部署或对外承诺前应明确：

- 顶部内容工具栏的后退、前进、自动换行和“更多操作”目前仍是界面占位；代码中未看到日志实时追踪（tail/follow）和当前文件刷新能力；
- 当前内置主题只有深色主题，旧版 `system/light/dark` 配置会归并到深色主题，但可以加载自定义 TOML 主题；
- 压缩日志超过 30 MiB 会自动写入 `~/.argus/cache/log_pages`，读取句柄释放后异步删除。当前分页缓存采用固定策略，不提供开关或容量上限配置；
- RAR 的目录、正文和加密兼容性取决于 RAR 家族、压缩方法和加密算法，不宜宣称支持所有 RAR 文件；
- SSH 密码、Git HTTPS 令牌、Git/SVN SSH 私钥口令、SVN 密码和 SMB 密码会随连接配置明文写入 `~/.argus/settings.toml`，但仓库秘密不会进入过滤结果或 Debug 输出。生产环境应限制该文件权限，后续建议接入系统钥匙串或凭据保险库；
- 远程文件管理当前不提供新建目录、跨目录移动或远程文件复制；删除目录仅支持空目录；
- Git/SVN 不支持仓库写入、目录递归下载、任意 Git 提交 SHA 输入、SVN 历史列表、子模块递归浏览或 Git LFS 对象下载；Git 缓存首版不设置自动容量上限；SVN HTTP(S) 要求服务端提供 Subversion 1.7+ HTTPv2 修订资源头并开放 `OPTIONS`、`PROPFIND`、`GET`，首版支持匿名和 HTTP Basic，不支持 Digest、NTLM 或客户端证书认证；
- Runtime 分析依赖项目约定的请求日志格式，并不是任意日志都能自动生成请求与 SQL 统计；
- 自动升级依赖用户自行配置可信升级服务器和 Ed25519 公钥；未配置时不会发起升级请求；
- 系统右键入口的注册管理目前面向 Windows 和 macOS，其他平台会返回不支持说明；
- 仓库包含 macOS、Windows 打包脚本以及 Linux 升级资产选择逻辑，但各目标平台的完整发布质量仍应通过对应系统的构建、安装和交互验收确认。
